## Ecommerce Detail Page Route

Default route:

1. Reset previous product context when the user says `生成新的详情页`; do not inherit prior product DNA, SKU colors, IP, artwork, packaging, claims, style, or correction history.
2. Analyze current product, source roles, style reference, and current user corrections.
3. Establish product DNA, package lock, claim-source categories, and unknowns from current inputs only.
4. Fully list `文案方向`, `风格方向`, and `画面方向`; stop for direction confirmation.
5. Produce a full screen-by-screen blueprint from the confirmed direction.
6. Build text master, visual master, product hard-constraint prompt, package prompt, and optional 1:3 spatial master.
7. Generate screen 01.
8. Generate screen 02.
9. Stitch a 2-slice preview when useful and audit continuity/style.
10. Present screen 01, screen 02, optional stitched preview, and audit; stop for style confirmation.
11. Generate remaining slices only after style confirmation, stitch the full long page, audit, revise, and deliver.

Do not treat separate screens as independent posters. Each slice should feel like a segment of one continuous ecommerce page system.

If the current request does not include usable product sources, stop and ask for the new product image, six-view, real photo, package file, or spec source. Do not use the last product as a fallback.

## Digital Product 11-13 Screen Order

Use this by default for Chinese ecommerce pages for earphones, power banks, speakers, cables, chargers, stands, fans, lamps, blow dryers, IP-style accessories, and small electronics.

1. **Hero / 首页主视觉**: product identity, strongest buyer reason, clear product subject, and restrained scene or brand atmosphere.
2. **Core Selling-Point Grid / 核心卖点宫格页**: fixed screen-02 module. Prefer a 2x3 six-point grid like the user's reference; use 5 cards only when there are fewer sourced or buyer-relevant points. Each card carries one buyer-facing point with a number badge, short title, short explanation, and visual proof such as product crop, structure proof, use thumbnail, icon, phone/device relation, backpack portability preview, or SKU/color proof. Keep unified card radius, spacing, glass/soft card language when suitable, and readable typography.
3. **Core Difference / 核心差异页**: explain the strongest reason it differs, such as portability, structure, charging convenience, stability, storage, or giftability.
4. **Structure Proof / 结构展示页**: show six-view logic, exploded view, macro detail, section view, hinge/port/line/slot/nozzle structure, or real assembly relation.
5. **Function Experience / 功能体验页**: show how the product is used, operated, opened, plugged in, folded, worn, placed, held, charged, or stored.
6. **Scenario / 场景应用页**: office, commute, travel, bedside, car, outdoor, study desk, live streaming, dorm, or gift scene.
7. **Detail Craft / 细节工艺页**: material, texture, port, button, anti-slip, hinge, line body, mesh, vent, nozzle, or package finish.
8. **Compatibility / 兼容适配页**: devices, placement, interface relation, phone/tablet/laptop/car/desk use. Avoid exact compatibility lists unless supplied.
9. **Safety Or Stability / 安全稳定页**: use only when supported by real facts. Show temperature, protection, load-bearing, certification, or stability without inventing numbers.
10. **Packaging And List / 包装清单页**: package, product, accessories, manual, cable, bag, nozzle, card, or gift set. Keep counts true.
11. **Parameter Table / 参数页**: use verified parameters only; unknown fields remain blank or "以实际参数为准".
12. **Comparison / 对比页**: optional. Compare with ordinary use, old solution, or same-series variant without false competitor claims.
13. **Closing / 收尾页**: restate the main memory point, brand mood, gift reason, daily-use value, and final CTA.

If the product has multiple confirmed colors or SKUs, include a color/SKU collection screen somewhere in the middle or near the SKU section. Use one angle, one light, one scale, and confirmed color names only.

If the user has limited information, keep the full order but reduce density. Do not fill missing screens with fake parameters.

## Buyer-Angle Selling Points

Plan selling points from buyer questions first for every product category, then attach proof level. Do not build the page from a raw parameter list.

Use this thinking sequence:

1. Identify the buyer: gift buyer, student, commuter, office user, collector, parent-child buyer, or practical daily user.
2. List what that buyer cares about in this product category.
3. Select 2-4 first-screen claim seeds from the strongest buyer concerns.
4. Classify each claim as confirmed, visible fact, reasonable inference, or unknown.
5. Turn unsupported performance claims into soft visual/copy directions instead of exact claims.

Category buyer concern map:

- **Earphones/headphones**: shape and IP appearance, wearing comfort, sound quality, battery/endurance, charging-case storage, connection/use convenience, portability, giftability, and daily scenario fit.
- **Power banks**: appearance/IP style, capacity/endurance, charging convenience, port/cable layout, screen/indicator clarity, pocket/bag portability, device compatibility, travel/gift value, and safety/protection when sourced.
- **Blow dryers**: appearance/IP style, wind strength, drying efficiency, hair-care comfort, temperature control when sourced, foldable storage, travel portability, noise impression, grip comfort, nozzle/accessory use, and giftability.
- **Phone/tablet stands**: appearance/IP style, support stability, viewing angle, foldability, phone/tablet compatibility, anti-slip contact, desk/bedside use, portability, and gift value.
- **Chargers/cables**: charging convenience, port type, cable storage, device compatibility, portability, durability feel, desk/travel use, and safety/protection when sourced.
- **Speakers**: appearance/IP style, sound impression, battery/endurance, volume/scene fit, portability, controls, gifting, and room/desktop atmosphere.
- **Fans/lamps/small appliances**: appearance/IP style, practical effect, comfort, battery/endurance when relevant, noise impression, portability, storage, safety/stability, and scene fit.

For any category, do not skip the buyer's core mental checklist. Earphones: "好不好看、戴着舒不舒服、音质如何、电池和续航够不够、连接/收纳是否方便、是否适合送礼". Blow dryers: "好不好看、风力够不够、干发是否方便、会不会伤发、能不能折叠收纳、旅行好不好带". Power banks: "好不好看、够不够用、充电方不方便、线口是否清楚、能不能放包里、是否适合出行". Stands: "稳不稳定、角度是否舒服、能不能折叠、是否适配手机/平板、桌面是否好用". Use exact performance numbers only when supplied.

Required visual modules for ecommerce product detail pages:

- Screen 01 must be a product-first hero. The product should be the first thing the viewer notices, with strong scale, clean lighting, clear silhouette, and minimal competing props.
- Screen 02 must be a fixed product-core-selling-point grid. It should quickly answer "what are the main reasons to buy" through 5-6 numbered cards, not a single poster composition. Prefer 2 columns x 3 rows for vertical Chinese detail pages. Use each card for one buyer concern, for example appearance/IP, comfort/use, performance, charging/power, portability, color/SKU, storage, compatibility, or giftability depending on category and proof level.
- Include at least one realistic model-use screen. This can be a hand-held, using, packing, drying, wearing, charging, placing, or operating scene depending on the product category. Keep anatomy and grip logic believable.
- Include at least one backpack portability screen. Show the product being placed into, partly inside, or beside a backpack/travel bag to prove compact storage or travel convenience. Keep product scale true and avoid hiding important structure.
- The model-use and backpack portability requirements may be separate screens. Do not merge them if doing so makes the message crowded or weak.
- A small backpack-portability card inside screen 02 can preview the point, but it does not replace the required dedicated backpack portability proof screen unless the user explicitly asks for a shorter page.

Layout diversity requirements:

- For 11-13 screens, use at least 8 distinct layout archetypes.
- For 8 screens, use at least 5 distinct layout archetypes.
- Adjacent screens must change at least two of these: product scale, product angle, copy form, card structure, background structure, prop density, image framing, or proof module.
- Do not use `big headline + subtitle + centered product` more than twice in one detail page.
- Do not repeat the same card grid rhythm in consecutive screens.
- Use the uploaded style/reference pages to extract layout variety, color pacing, sticker/callout language, and playful ecommerce energy. Do not copy exact text, commercial fonts, official IP wordmarks, character art, or unrelated product features.

## IP Style And Audience Fit

When a detail page is for an IP product, assign the IP style before writing the screen blueprint:

- Identify the confirmed IP and whether official artwork is supplied, visible, or only referenced as mood.
- Define target audience: gift buyer, student, office commuter, collector, family, youth trend buyer, or other buyer group.
- Choose scene language, palette, prop level, and layout density from the IP and buyer group. For Sanrio, Crayon Shin-chan, PINGU, Disney, tokidoki, Brunch Brothers, and similar IP products, 少女风, 可爱, giftable, and collectible styling can be the default when the target audience matches.
- Keep official artwork and logos only where supplied or visible on the product/package.
- Allow small glossy PVC-style figure props as secondary accents when they support the target audience, but do not let them dominate, cover the product, or imply included accessories.
- Use generic license-safe typography directions; do not request commercial fonts or imitate official wordmarks.
- Keep copy original and product-specific; do not invent official slogans, authorization text, or co-branded claims.

Add `ip_style_direction`, `target_audience_fit`, and `font_license_rule` to every screen that uses IP styling.

## Color Strategy

Build detail-page color from the product and IP, then expand it without changing product identity.

- **Base colors**: take from the real product, confirmed SKU colors, package, supplied IP artwork, or approved style direction.
- **Analogous colors**: add neighboring hues to make scenes and layouts richer. Use them for backgrounds, cards, shadows, soft props, ribbons, transitions, and section rhythm.
- **Contrast accents**: use only a small amount for stickers, arrows, badge corners, icon dots, separators, small labels, sparkles, or callout highlights.
- **Suggested balance**: base and analogous colors should dominate the page. Contrast accents should remain visually secondary, normally a small area of the screen rather than a large background.
- **Product safety**: never recolor the product body, SKU, logo, official artwork, package face, or accessory to fit the palette.
- **Readability**: text cards need enough contrast; decorative contrast colors must not sit behind dense body copy unless readability is clear.
- **IP safety**: do not use contrast accents to imitate protected wordmarks, official logos, or another IP's signature system.
- **Continuity**: vary analogous color proportions across screens, but keep the same color family so the long page reads as one set.

Examples:

- Pink product: use soft pink, rose, peach, cream, and a small mint/blue/yellow accent.
- Blue product: use baby blue, sky blue, lavender, white, and a small coral/yellow accent.
- Red-white IP product: use cream, soft red, pale pink, warm white, and a small cyan/mint accent.
- Yellow/orange playful IP product: use butter yellow, cream, peach, warm orange, and a small blue/green accent.

## Scene Rendering Route

Choose the scene-rendering route before writing the blueprint. The route controls whether a screen should feel like a product-rendered ecommerce stage or a realistic lifestyle/model scene.

### Low-Model Product-Led Route

Use this route by default for products where the buyer can understand the value mainly from product form, SKU color, portability, charging, support, storage, or desk use:

- Earphones/headphones.
- Power banks.
- Phone/tablet stands.
- Chargers, cables, adapters.
- Speakers.
- Compact fans, lamps, desktop accessories, small gift electronics.

Visual style:

- Use polished 3D/cartoon-rendered ecommerce scenes.
- Use clean pedestals, rounded cards, soft desk mats, shelf/room props, packaging, stationery, tote/backpack, and IP-matched decorative objects.
- Keep the product as the first visual subject; props stay lower contrast.
- Use product placement, height, scale, overlap, and front/back relationship to create eye movement. Do not rely on unnecessary drawn guide lines, arrows, glowing trails, or visual lines unless the selling point needs a real annotation.
- For SKU or color collection, stagger products through foreground/background and high/low display bases while keeping scale logic fair and structure consistent.

Model use:

- A hand/model-use screen is still required by the full detail-page contract, but it can be later in the page and can be a hand-held, wearing, packing, desk-use, or operation proof.
- The model should support scale and use, not dominate the page.

### Realistic Model Lifestyle Route

Use this route for products where the buyer needs to see the body, hand, hair, face, grip, or use posture to believe the value:

- Blow dryers.
- Beauty appliances.
- Wearable or body-use products when comfort or effect is the main proof.
- Any product where the user explicitly asks for model出镜 or strong lifestyle impact.

Visual style:

- Use a more photoreal lifestyle scene with believable room, vanity, bathroom counter, bedroom, dorm, or daily-use environment.
- Keep lighting soft and commercial, but make the product feel physically held and used.
- Preserve hand anatomy, grip, occlusion, scale, and product structure.
- The model is a supporting proof element. The product must remain visually dominant and readable.

Blow-dryer screen 01 rule:

- Default to a model出镜 first screen.
- Show the model holding the blow dryer with the arm extended forward toward the viewer.
- Use stronger perspective tension: product close to camera, model slightly behind, clear foreground/background depth, and a dynamic diagonal arm/product line.
- Make the dryer the immediate first focus. The outlet/nozzle, barrel, handle, buttons, IP artwork, fold joint, and cord/accessory state must not be hidden or distorted.
- Avoid fisheye distortion, oversized hand deformation, blocked product identity, toy-like proportions, or a flat catalog pose.

### Mixed Route

Use a mixed route when the page needs both lifestyle impact and product proof:

- Blow dryer: realistic model hero and use screens; product-rendered scenes for structure, folding, nozzle, packaging, SKU, parameter, and core-grid proof.
- Earphones/power banks/stands: product-rendered hero and core proof screens; later hand/model scenes for wearing, holding, packing, or operating.

Record `scene_rendering_route` in every screen blueprint as one of `product_render_cartoon_scene`, `realistic_model_lifestyle_scene`, or `mixed_route_support_screen`.

## Product Rendering, Angle, Scale, And Depth

Use a polished ecommerce-render style unless the user asks for raw real-photo styling: clean 3D/product-render look, soft rounded highlights, clear edge silhouette, shallow realistic contact shadow, high-key lighting, and light props that support the IP mood. The product should look engineered and real, not redesigned or turned into a toy prop.

Every screen must pair the selling point with the angle that best proves it:

- **Appearance, IP graphic, giftability**: front or 3/4 front hero angle. Make the product large enough to read the silhouette, logo, surface artwork, and color.
- **Opening, mouth, inner chamber, lid, port layout, storage relation**: top view or 45-degree overhead. Crop closer when the inside is the proof.
- **Thickness, fold direction, hinge, handle, strap, cable route, inlet/outlet, backpack fit**: side, rear, or diagonal angle. Keep the key side structure visible.
- **Material, texture, lock, button, mesh, nozzle, seam, charging port, anti-slip detail**: macro crop or magnifier module. The product part can be oversized because the buyer needs proof.
- **Operation, wearing, holding, packing, folding, plugging, opening**: hand/model angle. Let the hand prove scale and use, but do not block identity structures.
- **Portability**: product entering, partly inside, or beside a backpack/travel bag. The bag can be larger, but the product must remain readable and believable in scale.
- **SKU/color**: same product angle, same scale, same lighting, same shadow, and confirmed colors only.
- **Parameters or product information**: product becomes secondary and smaller; the information card/table is the proof.

Use product scale as page rhythm:

- Screen 01: product is the largest visual subject, usually 50%-75% of the screen area depending on crop. Product sits forward in depth.
- Screen 02: product/detail proof is smaller inside 5-6 cards. Information density is the hero.
- Structure/detail proof: product part can be very large or cropped; use close-up scale to prove detail.
- Scenario/model/backpack: medium product scale, normally 35%-70%, so the real context can explain the use.
- Packaging/list/SKU: keep consistent scale relationships; do not make variants different sizes unless the product truly differs.
- Parameter page: product can shrink to 20%-35%; text/table clarity leads.

Depth placement should explain the screen:

- Product in front, props behind: use for hero and core visual proof.
- Hand, bag, or action in front with product partly visible: use for real-use or portability proof.
- Large product behind with small detail cards in front: use for structure explanation.
- Multiple products staggered front/back: use for SKU or collection only when sizes remain comparable.
- Diagonal product crossing the screen: use for dynamic cute/IP energy and to avoid static poster repetition.

The advantage of this system is that each angle becomes evidence: front view sells appearance, overhead sells opening or layout, side view sells thickness/fold/fit, macro sells detail, hand/model sells scale, backpack sells portability, same-angle lineup sells color choice.

## Product-Background Visibility

Decide whether the background should make the product stand out or partially blend into the scene.

Use **stand-out treatment** for:

- Screen 01 hero.
- Screen 02 card proofs.
- Structure, interface, port, lid, inner chamber, material, macro, and parameter proof.

Stand-out methods:

- Lighter or cleaner background behind the product.
- White or translucent card backing.
- Clear contact shadow and rim light.
- Product edge contrast against the background.
- Small contrast accents for labels, arrows, badges, or numbers.
- Props lower in contrast than the product.

Use **partial-blend treatment** for:

- Lifestyle scene.
- Model/hand-use scene.
- Backpack/travel-bag portability.
- Gift display or desk scenario.

Partial blend methods:

- Shared light direction and color temperature.
- Analogous background colors from the product/IP palette.
- Believable contact shadow, occlusion, and perspective.
- Props that echo the product color but do not match it so closely that the product disappears.

Rules:

- A cute/IP background should make the product feel like it belongs to the IP world while the product remains readable.
- If product and background are both light blue, pink, cream, or white, add card backing, shadow, outline contrast, red/yellow/blue small accents, or a darker title/label zone.
- Do not use a high-saturation background area behind dense body copy.
- Do not let decorative props, PVC figures, stickers, ice blocks, balls, ribbons, or packaging become visually stronger than the product.
- For SKU pages, the background may follow SKU color, but the product body color must stay true and visible.

## Layout Diversity Patterns

Choose layout archetypes by screen job and buyer question. Use one coherent visual system, but vary the screen structures.

Recommended archetypes for cute/IP-style digital product detail pages:

- **Product-dominant hero**: oversized product, clear silhouette, compact headline, minimal props, first-screen only or opening recap.
- **Core selling-point grid**: fixed screen-02 5-6 card grid, preferably 2x3, with number badges, short card titles, short explanations, and visual proof per card.
- **Selling-point card collage**: 5-6 compact cards with icons, detail crops, mini scenes, or short labels. Use outside screen 02 only when it does not repeat the fixed screen-02 rhythm.
- **Sticker callout map**: product at an angle with arrows, labels, stamps, tape shapes, or small badge notes around real structures.
- **Macro magnifier proof**: large crop of port, lid, switch, texture, spout, hinge, screen, strap, or inside structure with a circular/rounded magnifier.
- **Hand/model operation**: realistic hand, model, or body-use composition showing open, pour, drink, hold, wear, pack, charge, or adjust.
- **Backpack portability scene**: product partly inside, entering, or placed beside a backpack/travel bag, with believable scale and visible key structure.
- **Photo-frame/polaroid stack**: overlapping photo cards, tilted frames, snapshots, or album-like panels for lifestyle or gift moments.
- **Split color block**: two-color or three-color background blocks separating product, copy, and proof modules.
- **Comic/speech-bubble panel**: playful speech bubbles, curved labels, exclamation marks, or story panels when IP mood supports it.
- **Step sequence**: numbered mini frames for open/close, use, cleaning, charging, storage, or carry steps.
- **Shelf/desk scene**: product placed in a desk, vanity, bedside, kitchen, dorm, office, or gift table environment.
- **Packaging/list grid**: package, product, accessories, manual, cable, strap, pouch, or sticker list with true counts.
- **Parameter sheet**: clean information card/table with confirmed parameters and placeholders for unknowns.
- **SKU/color collection**: all confirmed colors in consistent angle, scale, light, and spacing.
- **Closing mood panel**: quieter brand/product memory screen, gift atmosphere, daily-use value, and final CTA.

Suggested pacing for 11-13 screens:

1. Product-dominant hero.
2. Core selling-point grid.
3. Sticker callout map or split color block.
4. Macro magnifier proof.
5. Hand/model operation.
6. Scene shelf/desk or lifestyle environment.
7. Backpack portability scene.
8. Step sequence or comic/speech-bubble panel.
9. Packaging/list grid.
10. SKU/color collection or comparison.
11. Parameter sheet.
12. Photo-frame/polaroid stack or trust proof.
13. Closing mood panel.

Adapt the order to the actual product. The goal is varied information architecture, not forced decoration.

## Lightweight 8-Screen Order

Use this when the user explicitly asks for 8 screens, the product information is light, the page is not China-marketplace focused, or the subject is an AI tool/SaaS/developer product.

1. Hero: product identity and primary value.
2. Pain point or desire: buyer problem or current workaround.
3. Core benefit: strongest functional or emotional result.
4. Mechanism/detail: structure, feature, or proof.
5. Scenario: realistic use cases.
6. Trust/proof: verified facts, reviews, policy, or safe placeholders.
7. Comparison/value: variants, bundle, old-vs-new, or value stack.
8. Closing CTA: final reason to act.

Keep the first-screen claim seeds broad enough to support the later screens.

## Blueprint Fields

Each screen blueprint should include:

- `slice_id`: sequential id.
- `buyer_question`: the buyer question answered by this screen.
- `module_type`: internal section type, such as `hero`, `overview_cards`, `core_difference`, `structure_proof`, `function_experience`, `scenario`, `detail_craft`, `compatibility`, `safety_trust`, `package_list`, `parameter_table`, `comparison`, or `closing`.
- `module_label`: short internal job label. Do not write it as visible copy.
- `claim_seed`: one of the first-screen seeds this screen establishes or expands.
- `screen_job`: the page responsibility of this screen.
- `evidence_type`: visible evidence type, such as product identity, lifestyle result, macro proof, structure evidence, operation sequence, scenario caption, verified parameter, or closing emotion.
- `content_density`: `low`, `medium`, or `high`.
- `layout_archetype`: layout intent, not a rigid template. For screen 02, this must be `core_selling_point_grid`.
- `layout_variation_reason`: how this layout differs from the previous and next screen.
- `color_strategy`: base color, analogous extension, contrast accent color, and where each color appears.
- `copy_module_type`: visible-copy form, such as hero headline, guide title, Q&A, labels, annotation map, icon bullets, scene captions, parameter table, trust checklist, comparison snippets, or CTA.
- `copy_structure_pattern`: sentence/module rhythm, such as `hero_claim_stack`, `question_answer`, `overview_cards`, `annotation_map`, `three_point_breakdown`, `scene_caption_cluster`, `mini_steps`, `trust_checklist`, `parameter_rows`, or `quiet_closing`.
- `primary_module`: main visual and message.
- `secondary_modules`: 1-3 supporting modules when appropriate.
- `text_exact`: visible Chinese copy only.
- `text_layout_reference`: text arrangement borrowed from references, such as large playful headline, sticker label, arrow callout, speech bubble, bracket label, rounded card text, photo caption, mini comparison box, or parameter sheet.
- `hierarchy_strategy`: dominant text, secondary text, small labels, and spacing logic.
- `composition_shift`: how this screen differs from neighbors.
- `top_edge_anchor`: visual carryover from previous slice; no critical text here.
- `bottom_edge_anchor`: visual lead-in to next slice; no critical text here.
- `visual_composition`: product, packaging, props, model/hand, background, and module placement.
- `product_render_style`: product rendering language, light, shadow, material clarity, and whether it is product-render, photo-real scene, macro proof, or card-render proof.
- `scene_rendering_route`: category route for this screen, such as `product_render_cartoon_scene`, `realistic_model_lifestyle_scene`, or `mixed_route_support_screen`, with why this route fits the product and buyer.
- `product_angle_reason`: why the chosen product angle proves this screen's selling point.
- `product_scale_depth_reason`: product size, front/back placement, occlusion, and why that scale/depth supports the screen job.
- `product_background_relationship`: whether the product should stand out or partially blend into the background, and how readability is protected.
- `hero_title_layout_reason`: screen 01 only; why the title sits there, how it leads the eye to the product, and what it must not cover.
- `product_dna_lock`: relevant product structure constraints for this slice.
- `package_lock`: relevant package constraints or "not shown".
- `ip_style_direction`: IP mood, palette, scene, prop level, optional PVC-style figure accent rule, and what not to invent.
- `target_audience_fit`: which buyer preference the screen is serving.
- `font_license_rule`: generic safe typography direction and ban on commercial fonts/protected wordmarks.
- `claim_source`: confirmed / visible fact / reasonable inference / unknown placeholder.
- `reference_style_notes`: style DNA to borrow without copying brand, person, or unrelated product.
- `reference_layout_borrow`: specific layout language borrowed from references, such as sticker callouts, photo frames, macro magnifier, card collage, split color blocks, or step panels.
- `risk_unknowns`: unsupported claims, structure risks, text risks, or source gaps.

Before writing the blueprint, confirm the direction package. The direction package is a required user-facing output, not an internal note. Use these exact headings and fully list each section before stopping:

1. `文案方向`
2. `风格方向`
3. `画面方向`

- `copy_direction`: claim seeds, copy tone, headline style, key phrases, and banned claims.
- `buyer_angle_selling_points`: buyer type, category concerns, selected claim seeds, and proof level for each claim.
- `style_direction`: target audience, IP mood, palette, analogous color extension, contrast accent rule, lighting, typography, text layout reference, card/icon language, and prop density.
- `visual_content_direction`: hero focus, model-use idea, backpack portability idea, packaging/list idea, scene props, product angles, and layout diversity route.
- `project_context_status`: confirm this is a new project and previous product information has not been carried over, unless the user explicitly requested continuation.

Map the internal fields into the visible headings:

- `文案方向`: buyer-angle selling points, first-screen claim seeds, headline style, copy tone, key phrases, claim-source proof level, and banned/unsupported claims.
- `风格方向`: IP mood, target audience, product category tone, color palette, neighboring analogous colors, contrast accent limits, typography safety, scene-rendering route, prop density, and layout mood.
- `画面方向`: hero product focus, screen-01 title placement, screen-02 core grid plan, model-use scene, backpack portability scene, packaging/list scene, product angle logic, scale/depth rhythm, product-background relationship, and visible structure risks.

End the direction package with a confirmation line such as `请确认这三个方向，确认后我再生成完整蓝图。` Do not advance to the blueprint until the user confirms or revises these directions.

## Copy And Hierarchy Rules

Screen 01 may use the clearest headline/subtitle structure. Later screens should not all become new campaign openers.

Use varied copy forms:

- Strong hero headline for the opening.
- Fixed screen-02 core grid: 5-6 short numbered card labels, one short explanation per card, and visual proof per card.
- Small guide title plus annotations for structure/detail.
- Q&A or mini steps for function.
- Scene captions for use cases.
- Checklist or verified table for trust and parameters.
- Quiet emotional/brand copy for closing.

Avoid repeating the same promise with different words. If three consecutive screens can be summarized as `headline + one sentence + three tags`, rewrite at least two.

Keep title length restrained:

- Hero title: preferably within 12 Chinese characters.
- Short selling point: preferably within 20 Chinese characters.
- Overview card title: 4-10 Chinese characters.
- Overview card explanation: 15-20 Chinese characters.
- Parameter text: left-aligned and source-backed.

Do not display internal labels such as `module_type`, `screen_job`, `parameter_trust`, or `detail`.

## First Screen Title Rules

Screen 01 title is not only copy; it is the visual entry point. It should make the buyer feel the IP/product mood first, then lead the eye to the product.

Use these placement rules:

- **Top center**: use when the product is centered, symmetrical, front-facing, gift-like, or needs a cover-poster feeling.
- **Top-left**: use when the product sits right, tilts diagonally, or the layout needs a left-to-right reading path.
- **Slightly tilted or sticker-like**: use only when the IP mood is cute, comic, pop, or youthful. Keep readability higher than decoration.
- **Title wrapping near product**: use sparingly for cute IP pages; keep it near the product but not on key structures.
- **English decorative word plus Chinese title**: English can create mood, but Chinese carries the selling meaning.

Screen 01 title content:

- Prefer emotion or core buying reason over parameters.
- Keep it short, usually 4-12 Chinese characters for the main title.
- Use the product/IP/package palette; contrast color should be an accent, not the whole title block.
- Do not imitate official IP wordmarks or commercial display fonts.
- Do not cover product silhouette, front artwork, logo, port, button, handle, hinge, nozzle, fold line, case opening, or package face.

Screen 01 composition should read as:

1. Title creates the memory point.
2. Product becomes the first visual subject.
3. Short subtitle or small labels explain the reason to continue.
4. Bottom edge leads naturally into screen 02's core selling-point grid.

## Text Layout Reference Patterns

When the user supplies detail-page references, use them for text arrangement and information rhythm. Borrow the system, not the exact content.

Useful text-layout patterns from cute/IP-style ecommerce references:

- **Large playful headline block**: short 4-12 Chinese character headline, strong weight, compact subhead, placed near the main product without covering it.
- **Sticker label cluster**: small labels, stamps, tape tabs, rounded tags, or corner badges pointing to real selling points.
- **Arrow callout**: short label plus arrow/curve line pointing to a true product structure or use action.
- **Speech bubble / comic note**: playful small copy for IP-style pages; use sparingly and keep readable.
- **Bracket or boxed label**: square bracket titles, pill labels, or outlined text boxes for functional claims.
- **Mini card grid**: 2x2, 2x3, or vertical cards with one short title and one short explanation each.
- **Photo caption stack**: tilted photo/polaroid cards with short scene captions.
- **Macro annotation label**: detail close-up plus small technical label; exact technical claim requires source.
- **Comparison mini box**: two or three small comparison panels for use scenarios or variants, without false competitor claims.
- **Parameter sheet**: clean table or information card with confirmed data only; unknowns stay blank or "以实际参数为准".

Rules:

- Keep commercial typography safe: use generic rounded sans-serif, bold sans-serif, or soft generic handwritten-style descriptions.
- Do not name or imitate commercial fonts, official logo lettering, Coca-Cola script, Disney title style, Sanrio logotype, or any protected wordmark.
- Do not copy reference text, slogans, character titles, brand marks, barcode text, manufacturer text, or unrelated product claims.
- Vary text placement across screens: top-left headline, side label cluster, bottom card strip, floating sticker, centered parameter sheet, diagonal caption, and small scene labels.
- Keep text zones away from product ports, buttons, grilles, vents, handles, nozzles, fold joints, package faces, and key artwork.
- For AI-generated raster text, use short copy and plan manual replacement for final commercial accuracy.

## Continuity Rules

Use one visual system across the page:

- Same product model and SKU logic.
- Same background world or compatible material family.
- Same light direction, shadow, and product rendering style.
- Same typography family and hierarchy rules.
- Same text layout system, with varied placement and module rhythm across screens.
- Same IP style logic, target-audience scene language, and font-safety rule when IP styling is used.
- Same base palette logic, with controlled neighboring analogous colors and small contrast accents.
- Same icon/card language.
- Recurring motifs that transition rather than restart.

Every slice needs top and bottom edge behavior. Edge zones carry background, material, small props, ribbons, shadows, particles, desk/floor plane, or color flow. Keep main product proof and critical copy out of edge zones.

Fail continuity if each slice restarts with centered product, big headline, identical badges, and no visual carryover.

Use a 1:3 spatial master when the page needs strong continuity. Treat it as a spatial and hierarchy anchor, not a crop source or final image.

## Main, White-Background, SKU, And Packaging Sets

Use these only when the user asks for related image sets or the detail-page deliverable includes them.

For detail-page projects, related image sets are downstream of the final approved detail page. Finish the full detail page first, then derive square assets from the approved story, product DNA, and visual master.

Required downstream order:

1. Detail page: final slices, stitched long image, and audit.
2. Main images: `1:1` square images derived from approved detail slices.
3. SKU images: `1:1` square images generated after the main-image set.

Do not generate SKU images before the detail-page set and main-image set are complete.

### Main Image Set From Detail Pages

- Count: match the final detail-page slice count. An 11-screen detail page produces 11 square main images; a 13-screen detail page produces 13 square main images.
- Size: `1:1` square, default 1400 x 1400 unless platform differs.
- Method: recompose each approved detail slice into a square main image. Preserve the slice's message, product angle, scene style, and copy hierarchy.
- Do not crop a long slice, stretch it, squeeze it, or remove key proof modules only to fit square size.
- Keep the product as the main visual subject in every square image. For screen 01-derived main image, product focus must be especially strong.
- Simplify long-page edge anchors when converting to square; square images do not need vertical continuation cues.
- Keep typography license-safe and avoid commercial font names or protected IP wordmarks.

### White-Background Set

- Size: 1400 x 1400 square unless platform differs.
- Background: pure white `#FFFFFF`.
- Shadow: natural light-gray contact shadow.
- Product size: 75%-85% of canvas.
- No props, complex background, or large text.
- Generate one product-only image per confirmed color/SKU.
- Generate package-only image per package version.
- Generate product + package image per confirmed color.
- Optional state/accessory white-background images only for real states/accessories.

### Main Image Set

Use this lighter set only when the user asks for a compact main-image package instead of converting every detail slice:

- Core selling point.
- Real use scene.
- Function or operation.
- Detail/material/structure.
- Packaging/gift/list.
- SKU or all-color collection.

Keep one product identity and one visual family.

### SKU Set

- Timing: generate after the final detail page and the `1:1` main-image set.
- Size: `1:1` square, default 1400 x 1400 unless platform differs.
- Count: match confirmed product colors. One confirmed color produces one SKU image; three confirmed colors produce three SKU images.
- Use only confirmed colors/SKUs.
- Default square scene layout.
- Left side: color/SKU name and 5-9 short selling points.
- Right side: corresponding product in true color and structure.
- Background supports SKU color but does not overpower the product.
- Add a collection SKU only when multiple real colors exist.
- If colors are not confirmed, ask once or mark SKU count as pending; do not invent colors from IP mood or style references.

### Packaging/List Set

- Keep package type, face ownership, proportions, and accessory count true.
- Reserve small official text, barcode, certification, anti-counterfeit, authorization, and manufacturer areas for official compositing.

## SaaS, AI Tool, And Developer Product Pages

For AI tools, model pages, SaaS features, plugins, and developer products, use product-detail-page logic without the digital-product structure constraints.

Recommended AI/model/tool order:

1. Product identity and one concrete user outcome.
2. Visual proof, examples, generation previews, or workflow screenshots.
3. Capability map.
4. Use cases from the user's perspective.
5. Controls, modes, limits, formats, quality settings, or safety constraints.
6. Developer/operator details when relevant.
7. Comparison or upgrade path.
8. CTA.

Recommended SaaS order:

1. Feature name and operational promise.
2. Before/after workflow.
3. Key screens, tables, dashboards, or automations.
4. Roles and permissions.
5. Integrations and data flow.
6. Metrics, reporting, auditability, or reliability.
7. Adoption CTA.

Recommended developer-product order:

1. Product/API name and what developers can build.
2. Quick code or request/response example.
3. Supported platforms, SDKs, models, formats, or endpoints.
4. Reliability, rate limits, auth, pricing-relevant constraints, and security.
5. Real integration examples.
6. Documentation or sandbox CTA.

Keep operational pages dense, restrained, and concrete.

## Review Rubric

Score each area as pass, needs work, or missing:

- Product DNA is explicit and used in every relevant screen.
- Previous product information is not present when the request is a new detail-page project.
- Source roles are separated correctly.
- IP style is matched to the target audience without adding unauthorized characters, logos, slogans, or commercial font names.
- Copy, style, and visual content directions were confirmed before the blueprint.
- Unsupported parameters and claims are absent or marked.
- First viewport clearly says what the product is and why it matters.
- Claim seeds are chosen from buyer concerns rather than a raw feature list.
- First viewport makes the product visually dominant and immediately noticeable.
- Screen 01 title placement supports the product-first hero and does not cover product identity structures.
- The page includes at least one model-use screen.
- The page includes at least one backpack portability screen.
- Screen 02 is a real 5-6 point overview when using the digital-product route.
- Screen 02 uses the fixed core selling-point grid structure, preferably 2x3, with number badges, short titles, short explanations, and visual proof in every card.
- Each screen answers a different buyer question.
- Each visual screen uses an angle, scale, and depth relationship that proves its selling point.
- Product-background relationship is intentional: hero/detail screens make the product stand out, while lifestyle scenes may partially blend without losing readability.
- Copy patterns vary across adjacent screens.
- Text layout references are used as arrangement language without copying exact reference text or protected wordmarks.
- Layout archetypes are diverse enough for the selected screen count.
- Adjacent screens do not repeat the same product scale, copy rhythm, and card structure.
- Color strategy uses product/IP base colors, neighboring analogous colors, and only small contrast accents.
- Contrast accents do not overpower the product or damage readability.
- Product structure, SKU color, packaging, and accessories remain consistent.
- Main-image count follows final detail-page slice count when the project requires post-detail main images.
- Each detail slice is recomposed into a readable `1:1` square image without cropping, stretching, or squeezing.
- SKU images are generated after detail pages and main images, and use `1:1` square format.
- SKU image count follows confirmed product colors only.
- Visual system is unified across the whole page.
- Top and bottom edge anchors support long-page continuity.
- Text is readable and not garbled.
- Final stitched preview reads as one detail page, not separate posters.
