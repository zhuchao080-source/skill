# zhuchao的SKILL

zhuchao的SKILL 是一个用于规划、生成、审核和交付电商详情页的 Codex Skill。调用语为：`生成新的详情页`。它适合把产品图、六视图、包装图、参数资料和风格参考，转成一套结构稳定、叙事连续、文案克制的电商详情页流程。

新版规则根据数码产品详情页和结构保真规则优化：先建立产品 DNA 和结构锁定，再做页面规划、视觉母版、分屏生成、拼接审核和局部返修。

## 核心能力

- 分析产品图、六视图、包装图、说明书、参数表和风格参考
- 调用 `生成新的详情页` 时自动清空上一个产品信息，不继承旧产品 DNA、颜色、IP、卖点、包装或参数
- 区分已确认信息、图片可见事实、合理推断和未知参数
- 所有产品卖点都先从买家角度思考，再对应到产品事实和可证明信息
- 先建立产品 DNA，锁定外形、接口、按键、屏幕、线材、转轴、包装和配件
- 根据产品 IP、产品调性和对应人群共同决定场景、配色、道具、目标人群和文字排版
- 场景风格和详情排版可在主色基础上加入邻近色，并用少量对比色做点缀
- 三丽鸥、蜡笔小新、PINGU、迪士尼、tokidoki、早午餐兄弟等 IP 可按目标人群走少女风、可爱风、礼赠风或收藏展示风
- 中国电商数码产品默认规划 11-13 屏详情页
- 用户明确要求时仍可规划 8 屏轻量详情页
- 第二屏固定为产品核心卖点宫格页，优先 2x3 六宫格，每格包含编号、短标题、短解释和对应证据图
- 每个卖点必须匹配能证明它的产品角度、产品大小、前后层次和背景关系，并说明为什么这样摆
- 首屏标题必须解释为什么放在该位置，标题要先制造记忆点，再把视线引到产品，不能遮挡产品关键结构
- 配色要判断产品是在背景里突出还是融入；首屏和结构页突出产品，场景页可适度融入但不能丢失轮廓
- 详情页排版必须多样化，避免每屏都是大标题、居中产品和重复卡片
- 文字排布和排版风格可参考用户提供的详情页：大标题、贴纸标签、箭头标注、气泡、卡片、参数表等
- 建立文字母版、视觉母版、产品结构锁定和可选 1:3 空间母版
- 生成逻辑固定为：先确认文案/风格/画面内容方向，再生成蓝图，再第一屏、第二屏，确认风格后再生成剩余分屏
- 详情页至少包含一张模特使用图、一张产品放入背包的便携卖点图
- 首屏必须让人一眼注意到产品，重点展示产品主体
- 检查产品漂移、文字混乱、参数编造、结构错误和页面连续性
- 详情页生成完成后，可按详情页屏数生成对应数量的 1:1 主图
- SKU 图在详情页和主图完成后生成，尺寸与主图一致为 1:1
- 支持同一项目下的主图、白底图、SKU 图、包装清单图规划
- 支持三丽鸥、蜡笔小新、PINGU、迪士尼、tokidoki、早午餐兄弟、可口可乐等 IP 风格方向
- PINGU 按其他 IP 一样建立完整风格档案：浅蓝/天空蓝/baby blue 做主基底，可撞奶油黄、暖白、红橙、粉、紫或黑白中性色；用纸片标题牌、胶带、贴纸、圆形台柱、前后层次和生活道具做出 MarTUBE 式日常电子/礼品感；不要绿色、薄荷绿、青绿色、草绿色、大面积深蓝、冰雪或空展台；不复制官方素材、字标、标语或未经确认参数
- PINGU 每屏最多只能出现一个额外 PINGU 场景形象，产品机身、包装、标签或屏幕里的印刷图案不计入；首屏和功能证明页优先少放或不放额外 PINGU 形象
- 可在场景中适当加入小型亮面 PVC 材质公仔点缀，但不能抢产品主体或暗示未确认配件

## 使用方式

最简单的请求：

```text
图1是产品六视图，图2是产品实拍，图3是风格参考。
生成新的详情页。
```

也可以补充产品信息：

```text
产品名称：SonicAir H7 Pro
产品品类：蓝牙耳机
核心卖点：舒适佩戴、桌面收纳、礼赠包装
真实参数来源：以说明书为准
禁止出现：主动降噪、防水、未经确认的认证
IP 风格：三丽鸥，面向礼赠和日常可爱桌面场景
可选道具：小型亮面 PVC 材质公仔点缀，作为次要背景道具
文字规则：不使用商用字体，正式 IP 字标用官方素材或安全可编辑文字层
目标平台：淘宝详情页，790px 宽
```

如果没有提供卖点或参数，zhuchao的SKILL 会使用产品图片可见事实和谨慎的品类推断，但不会虚构容量、功率、蓝牙版本、防水等级、认证、授权编号、制造商信息、条码或包装小字。

## 工作流程

1. **清空旧项目**
   调用 `生成新的详情页` 时，默认这是一个全新产品项目。不得继承上一个产品的结构、颜色、SKU、IP、包装、卖点、参数、文案、风格、文件名或修改记录，除非用户明确说继续同一个产品。

2. **分析输入**
   区分当前请求里的结构参考、表面图案参考、包装参考、风格参考和用户最新修正。如果当前请求没有产品图片或产品文件，先要求补充，不使用上一个产品代替。

3. **建立产品 DNA**
   锁定产品外轮廓、比例、接口、按键、屏幕、指示灯、LOGO、线材、转轴、材质分区、颜色分区、包装结构和配件清单。

4. **确认方向**
   先给出买家角度卖点、文案方向、风格方向和画面内容方向。风格方向必须先判断 IP、产品品类调性和对应人群是否统一，再确定目标买家、品类关注点、核心卖点、标题语气、IP/目标人群风格、首屏产品视觉、模特使用图、背包便携图和禁用内容。

5. **输出蓝图**
   用户确认方向后，再生成完整屏幕蓝图；每屏包含消费者问题、页面任务、文案结构、文字排布参考、视觉模块、内容密度、版式类型、版式变化原因、产品渲染风格、产品角度理由、产品大小和前后层次理由、产品与背景的突出/融入关系、结构锁定和风险未知项。第一屏还要写清标题为什么这样摆。

6. **建立母版**
   锁定文字母版、视觉母版、产品结构提示词、包装锁定规则和可选 1:3 空间母版。
   如果是 IP 产品，同时锁定 IP 风格母版、目标人群、场景道具和安全字体规则。

7. **生成第一屏**
   先生成详情页第一屏，用来确认产品主视觉、核心卖点、首屏氛围和文字层级。第一屏必须让产品一眼被注意到，产品是画面绝对主体。

8. **生成第二屏**
   再生成详情页第二屏，固定做产品核心卖点宫格页。优先使用 2x3 六宫格；每格必须包含编号、买家角度短标题、短解释和对应视觉证据，如产品局部、结构证明、使用缩略图、设备关系、便携预览或颜色/SKU 证明。

9. **确认风格**
   用户确认第一屏、第二屏和可选拼接预览的整体风格后，再生成剩余分屏。

10. **生成剩余分屏**
   保持同一产品模型、同一视觉系统和不同的信息任务。剩余详情页中必须包含至少一张模特使用图，以及一张产品放入背包或旅行包中的便携卖点图。11-13 屏详情页至少使用 8 种不同版式，8 屏详情页至少使用 5 种不同版式，相邻屏不能重复同一套视觉语法。

11. **生成主图**
   详情页完整生成、拼接和审核通过后，再根据最终详情页屏数生成对应数量的 `1:1` 主图。每张主图来自对应详情分屏的卖点和画面逻辑，但要重新排版为正方形，不能直接裁切、拉伸或压缩长图。

12. **生成 SKU 图**
   主图生成完成后，再生成 SKU 图。SKU 图尺寸与主图一致，必须是 `1:1` 方图；数量按确认颜色决定，几个颜色就几个 SKU。

13. **最终审核与交付**
   拼接完整长图，检查结构、文字、参数、SKU、包装和连续性；只返修受影响的单屏或相邻屏幕。

## 安装

### 让 Codex 安装

向 Codex 提出：

```text
请从 https://github.com/zhuchao080-source/skill 安装这个 Skill。
```

### 手动安装

Windows PowerShell：

```powershell
git clone https://github.com/zhuchao080-source/skill.git "$env:USERPROFILE\.codex\skills\detail-flow"
```

macOS / Linux：

```bash
git clone https://github.com/zhuchao080-source/skill.git ~/.codex/skills/detail-flow
```

安装后可以通过调用语 `生成新的详情页` 触发它。为保持 Codex Skill 校验通过，内部技能名仍保留为 `detail-flow`。

- `references/ip-style-guidance.md`：IP 风格、人群偏好、场景生成、文字排版和字体授权安全规则

## 设计原则

- 详情页是一条完整购买叙事，不是多张互不相关的海报。
- `生成新的详情页` 必须作为新产品项目处理，旧产品信息默认清空。
- 没有当前产品来源时，不能用上一个产品信息补齐，应要求用户提供当前产品图片或文件。
- 产品结构优先于美观效果。
- 风格参考只控制氛围、配色、光线和构图，不能覆盖产品结构。
- 风格必须同时符合 IP、产品自身调性和目标人群，不能只套统一可爱风；场景要既有 IP 情绪，也能说明产品真实用途。
- PINGU 方向按其他 IP 一样先建立完整风格档案：浅蓝/天空蓝/baby blue 为主基底，按 SKU、品类和人群撞奶油黄、暖白、红橙、粉、紫或黑白中性色；首屏突出产品并使用纸片标题、贴纸、台柱、道具和前后层次，第二屏用宫格证明核心卖点；避免绿色、大面积深蓝、冰雪化和空展台。
- PINGU 场景形象控制为每屏最多一个额外 PINGU 形象；产品上、包装上、标签上的 PINGU 图案不计数。
- 配色以产品/IP 主色为基础，允许添加邻近色丰富背景、卡片、道具和转场层次。
- 首屏和结构证明页要让产品从背景里跳出来；生活场景、手模、背包便携页可以让产品适度融入场景，但必须保留清晰轮廓、高光、阴影和关键结构。
- 对比色只能少量用于贴纸、箭头、徽章、图标点、分隔线或小装饰，不能抢产品主体，不能改变真实 SKU 颜色。
- Skill 覆盖的 IP 按已授权项目处理，可使用 IP 名称、角色世界、授权素材、包装节奏、官方风格标题层级和可见产品图案；但不得虚构授权编号、版权小字、条码、制造商信息、未确认联名、未确认赠品或未经来源确认的参数。
- PVC 公仔只能作为小型场景点缀，不得遮挡产品结构，不得被写进配件清单，除非用户确认它属于套装配件。
- 文字排版使用通用安全字体方向，不使用需商业授权的字体名称；IP 标题/字标可按授权项目规划，但正式落版优先使用官方素材或安全可编辑文字层。
- 文字排布参考详情图的结构语言，如大标题块、短副标题、贴纸标签、箭头标注、漫画气泡、括号标签、圆角信息卡、拍立得说明、对比小框和参数表。
- 首屏标题是视觉入口，不是参数列表；授权 IP 首屏优先建立“IP 大标题/标题区 + 中文产品卖点副标题”的层级，标题位置要跟产品位置形成阅读路线，常用顶部居中、左上错落、轻微倾斜或贴纸式标题，但不得抢产品主体。
- 文字参考只借鉴排布和层级，不复制参考图原文、商用字体、无关卖点、法务小字或未经确认参数；授权 IP 标题风格可以参考，但正式字标应使用官方素材或可商用文字层。
- 包装小字、认证、条码、授权编号和参数不交给 AI 猜写。
- 卖点必须从买家关心的问题出发，所有产品都一样先看目标买家在意什么，再写卖点。耳机考虑造型/IP 外观、佩戴舒适、音质、电池、续航、连接/收纳便利、便携和送礼；吹风机考虑造型、风力、干发效率、护发舒适、折叠收纳、便携和风嘴；充电宝考虑外观、够不够用、充电便利、线口布局、放包便携、出行和安全；支架考虑稳定、角度、折叠、适配和桌面使用。
- 第二屏卖点总览固定为核心卖点宫格，要帮助用户快速理解“为什么值得买”，不能做成单一大图或重复首屏主视觉。
- 颜值/IP 用正面或 3/4 角度，开盖/内部/接口用俯视或 45 度俯视，厚度/折叠/放包用侧面或后侧角度，材质/接口/锁扣用微距，使用体验用手模或模特，多色 SKU 用同角度同大小并排。
- 参考图只借鉴版式变化、色彩节奏、贴纸标注、卡片组合、微距放大、拍立得叠图、漫画气泡和场景层次，不复制原文字、商用字体、官方字标、角色图或无关卖点。
- 详情页要组合多种版式：大产品首屏、卖点卡片、贴纸标注、微距放大、手模/模特使用、背包便携、分色块、步骤图、包装清单、参数表和 SKU 集合等。
- 生图前必须先确认文案方向、风格方向和画面内容方向，再进入蓝图。
- 首屏产品必须是第一视觉中心，不能被模特、包装、公仔或道具抢主体。
- 详情页必须包含模特使用图和背包便携图，除非用户明确取消。
- 主图必须在详情页完成后派生，数量跟随最终详情页屏数，每个详情分屏对应一张 `1:1` 主图。
- SKU 图必须在详情页和主图都完成后生成，尺寸与主图一致为 `1:1` 方图。
- SKU 图数量必须跟随确认的产品颜色数量，不根据 IP 氛围或风格参考虚构颜色。
- 主图、白底图、SKU 图和详情页必须引用同一产品 DNA。
- 审核和局部返修是正式流程的一部分。

## 当前限制

- 图像模型可能产生产品结构或文字细节漂移，需要审核与局部返修。
- 1:3 母版用于统一视觉和叙事，不应直接裁切为最终分屏。
- 独立生成的分屏无法保证像同一张照片一样完全无缝连接。
- 精确参数、认证、授权和包装正式文字必须由用户提供可靠来源。
